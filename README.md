Emotion-Tracker-Lite基于离线架构与多维加权算法的心理预警系统项目愿景：在极致的物理约束（10MB 存储上限）与隐私保护（全离线架构）下，利用数学建模实现个体情绪波动的精准量化与分级预警 。  核心技术亮点 (Core Highlights) 极致隐私安全：采用全离线架构设计。系统不依赖任何外部服务器或云端数据库，用户的心情分值、压力标签及随笔数据“绝不出手机”，从物理层面上杜绝隐私泄露风险 。  多维加权量化模型：不同于传统的感性记录，系统引入压力源动态加权引擎。针对“重大变故”、“持续高压”、“日常琐事”等不同属性的压力源赋予差异化权重系数（W），将模糊的心理波动转化为具备数理参考价值的线性趋势指数 。   极客级存储优化：深度适配微信小程序 10MB 本地存储空间限制。底层封装异步读写操作，并实现基于 LRU (Least Recently Used) 策略的缓存淘汰算法，确保系统在有限空间内实现长效、无感知的稳定运行 。   高内聚工程实践：项目严格遵循 3NF (第三范式) 进行底层数据库建模 ，并在逻辑层实现线性插值算法，以自动补全用户断层数据，保证情绪量化曲线的连续性与科学性 。   系统架构说明 (Architecture)代码段graph TD

   技术栈与实现细节 (Implementation)前端框架：微信小程序原生框架 (WXML/WXSS/JavaScript) 。  可视化方案：基于 Canvas API 的响应式趋势图渲染 。  数据模型：采用逻辑解耦的映射表设计，支持 M:N（记录-标签）关系的高效查询。核心逻辑：插值算法：$V_t = V_{prev} + \frac{V_{next} - V_{prev}}{t_{next} - t_{prev}} \times (t - t_{prev})$，解决数据断层导致的分析失真。运维策略：周期性触发存储空间监控，自动淘汰陈旧无效数据 。   未来路线图 (Roadmap)[ ] 接入本地轻量级 NLP 模型，实现日记文本的自动情感极性识别 。  [ ] 开发脱敏数据导出功能，对接专业心理咨询标准接口 。
