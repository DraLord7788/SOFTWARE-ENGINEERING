📊 大学生个人情绪量化预警系统 (Emotion-Tracker)
基于多巴胺预测误差（RPE）与本地冷热数据分离架构的轻量级心理干预程序。

💡 项目背景与核心愿景
针对当代大学生面临的多维压力，市面上的心理打卡软件往往因为强制云端同步而引发用户的“隐私焦虑”。本项目立足于绝对数据主权与克制干预理念，旨在提供一个纯本地运行、高隐私安全的情绪趋势量化工具。系统不盲目提供医疗诊断，而是通过量化“情绪落差”来引导用户进行自我救赎与线下求助。

🚀 核心架构与技术创新 (Core Features)
本项目突破了传统“流水账式”记录的局限，在底层架构上实现了以下创新：

M1 模块 | 数据容错与插值降噪： 针对用户漏打卡导致的断层问题，采用线性插值算法自动拟合缺失值，保证情绪趋势曲线的连续性。

M2 模块 | 动态基准线与 RPE 修正： 引入神经科学中的多巴胺奖赏预测误差（Reward Prediction Error）逻辑。系统自动锚定用户的心理预期基准线，精准放大情绪落差感，实现比绝对阈值更敏锐的动态预警。

M3 模块 | 阻塞式危机干预： 当加权压力指数跌破安全阈值时，系统触发阻塞式弹窗，并在本地硬编码注入心理援助资源。

M4 模块 | 极客级存储运维： 面对微信小程序严格的 10MB 本地 Storage 限制，引入 LRU 淘汰策略 与 冷热数据分级架构。彻底解决存储溢出隐患。

🛠️ 部署与运行指南 (Quick Start)
本项目开发基于微信小程序原生框架，请评阅老师按照以下步骤运行代码：

环境准备： 请下载并安装最新版的微信开发者工具。

导入项目： 打开工具，选择导入项目，目录指向本源码的根文件夹。

AppID 设置： 考虑到测试环境，请在导入时选择 测试号 运行，避免因未配置合法域名导致的接口报错。

编译预览： 点击顶部菜单栏的编译按钮，即可在左侧模拟器中体验完整业务流。

🔮 未来演进 (Future Work)
在下一阶段，计划接入纯端侧轻量化大语言模型（On-device SLM），通过深度脱敏的特征哈希值，实现对诸如“补偿性消费”、“亲密关系波动”等具体行为的耦合分析与前置预测。
